The Sample Code and Files on CD-ROM

The sample code on the companion CD-ROM is organized into two sections. One section is for all Windows NT Visual C++ code and the other is for UNIX code. All the code has been compiled and tested on Windows NT 4.0 and Sun Solaris 5.6 (UNIX System V) machines. The compiler used on Windows NT is Visual C++ v5.0, and Sun's C++ compiler was used on the UNIX platform. It is the authors' belief that all code will successfully compile and run on all versions the operating system and compiler equal to or greater than those listed above. If you encounter any problems executing the code examples on future operating systems and/or compiler versions, please let the authors know (we would be even more interested in solutions to problems encountered). The authors can be contacted at unixnttalk@yahoo.com .

The CD-ROM listings are organized by chapter and section within a given chapter. When creating projects using Visual C++, a directory is created for each project. Therefore a typical directory structure for a Windows NT project is as follows.

chapterX\sectionY\projectZ

Because an integrated development environment is not used on UNIX, the projects are located directly within the chapter or section directory. If the chapter or section contains only one main project, then it would be found in its given chapter or section directory with a single Makefile . If multiple projects are associated within a chapter or section, then each project has a separate Makefile related to building the given project. The Makefile is named makefile_<project name> . To build the UNIX project, execute the following command at the command line.

make -f <makefile name>

Note: All example code used in this book uses objects and classes developed in our companion book Making UNIX and Windows NT Talk. None of the examples presented here would compile or link unless at least the header files and object modules from that book were included on this CD-ROM. We have elected to include all source and object code from Making UNIX and Windows NT Talk on the CD-ROM for this book, as doing so will aid the developer when inserting these components in his or her program. An unfortunate consequence of this decision was that naming clashes occur between the directories and chapters -- both books have Chapters 1, 2, and so on. Does the chapter1 directory on the CD-ROM refer to Chapter 1 in this book or Chapter 1 in Making UNIX and Windows NT Talk? We resolved this by sequencing the directories for chapters in this book after the directories for Making UNIX and Windows NT Talk and provide a map to convert the chapters in the book to the directories on the CD-ROM. The map is See Filenaming Conventions for the CD-ROM.. For example, to view the code for Chapter 3 of this book, look in the directory named chapter7 on the CD-ROM.

All code related to a particular chapter and section is located directly within that chapter and section. One of the goals of this book is to present each chapter and section as a building block for the next. Therefore, code in later chapters and sections use header and object files from applications in previous chapters and sections. To make things more convenient when compiling the Windows NT applications, commonly used header and object files are moved to their related chapter or section directory to avoid having to specify a particular project.

Compiling and Executing the Example Applications

When placing the sample code on a local machine, the directory structure must stay intact for the compiles to be successful. All UNIX projects contain a Makefile . The Makefile references header files, compiles source code, and links object code based on a directory relative to the one specified by the ROOT_DIR environment variable. Therefore, the first step in compiling the UNIX applications is to set the ROOT_DIR environment variable to the top-level directory where the UNIX code is located. If the UNIX code is placed in the /home/book directory, then ROOT_DIR would need to be set to /home/book .

The Windows NT projects contain project workspace files denoted by <project name>.dsw . To open a project in Visual C++, choose File - Open Workspace from the main menu and select the project. Compiling the application is simply a matter of choosing Build - Build from the main menu. Needed header and object files from previous chapters and sections are included within the project's setting specifications. The header and object files are linked relative to the current project; therefore, it is important that the local directory structure remain the same as on the CD-ROM.

Before executing the example applications, it is important that both the UNIX and Windows NT workstations are connected on the same network and each can be accessed by the other. When executing systems that have both a client and a server application, the server application must always be executed first unless the client automatically spawns its server.
